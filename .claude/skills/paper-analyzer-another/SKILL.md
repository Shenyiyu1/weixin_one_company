# Academic Paper Analyzer - 学术论文深度解析

## 核心能力

- **MinerU Cloud API** 高精度 PDF 解析
- 自动提取图片、表格、LaTeX 公式
- **论文配图强制嵌入**：架构图/实验结果/消融对比必须在文章中展示
- **多种写作风格**：故事型 / 学术型 / 精炼型
- **可选公式讲解**：插入公式图片并详细解读
- **可选代码分析**：结合 GitHub 开源代码讲解
- 输出 Markdown + HTML（base64 嵌入图片）

## 前置准备

### MinerU API Token

1. 访问 https://mineru.net 注册账号
2. 获取 API Token
3. 设置环境变量（推荐）：
   ```bash
   export MINERU_TOKEN="your_token_here"
   ```

### 依赖安装

```bash
pip install requests markdown
```

## 操作步骤

### 第一步：PDF 解析（使用 MinerU API）

```bash
python scripts/mineru_api.py <pdf_path> <output_dir>
```

或者直接传入 token：
```bash
python scripts/mineru_api.py paper.pdf ./output YOUR_TOKEN
```

**输出结果：**
- `output_dir/*.md` - Markdown 文件（含公式、表格）
- `output_dir/images/` - 高质量提取的图片（架构图、实验结果、消融对比等）

#### 1a. 建立 Figure→图片 映射表（必须）

MinerU 提取的图片文件名是 hash（如 `39a4ef3b...145d.jpg`），必须从 markdown 中找出 Figure 编号与文件名的对应关系：

```bash
grep -E "(Figure \d+|!\[\]\(images/)" output_dir/*.md
```

建立映射表：
```
Figure 1 | Architecture overview  →  images/39a4ef3b...145d.jpg
Figure 2 | Position-wise acceptance →  images/310bab3...c87e.jpg
Figure 3 | Depth ablation         →  images/a1b2c3d...7890.jpg
...
```

> ⚠️ **全部映射，不遗漏。** 这是后续文章中嵌入配图的基础。没有这个映射表，图片就是一堆无意义的 hash 文件名。

#### 1b. MinerU 图片分类策略（决定哪些图进文章）

MinerU 提取的图片不止有论文配图（Figure），还有公式片段、小表格、装饰元素等。不能全部无脑塞——也不能只挑 Figure 编号的图。按以下优先级分类：

| 优先级 | 图片类型 | 特征 | 处理方式 |
|--------|---------|------|---------|
| **必须** | 架构图 / 方法图 | 有 Figure 编号，展示系统结构或方法流程 | **全文嵌入**，放对应章节开头 |
| **必须** | 实验结果图 / 消融对比图 | 有 Figure 编号，含折线/柱状/表格对比 | **全文嵌入**，放实验段落 |
| **重要** | 公式片段 | MinerU 将 PDF 中公式渲染为独立图片 | **选择性嵌入**，次要公式可转文字描述。关键公式优先用 LaTeX 代码渲染，MinerU 公式图片作为复杂公式的备选 |
| **可选** | 表格图片 | MinerU 将复杂表格渲染为图片 | **选择性采纳**：数据表格转文字表格，合并单元格/复杂布局的表格可保留为图片 |
| **跳过** | 装饰元素 / 小图标 | 无信息量的小图、页面装饰 | 忽略 |

**操作流程**：
1. 先执行 1a 的映射表，标记所有 Figure 编号的图
2. 剩余的 hash 图片用 Read 工具抽检 3~5 张，判断类型（公式/表格/装饰）
3. 公式片段：对比论文中公式列表，有编号的重要公式→嵌入，无编号的一行公式→文字描述
4. 复杂表格（合并单元格多、宽表）：保留为图片嵌入；简单数据表：转 markdown 表格

> ⚠️ **公式渲染优先级**：**优先使用 LaTeX 代码渲染公式**（KaTeX / render-formulas.py），MinerU 公式图片作为备选。对于含 `\tag`、`\begin{cases}` 等复杂结构的公式，尽量用 LaTeX 书写——`render-formulas.py` 的 `\tag{}` 兼容问题应在脚本侧修复（检测到 `\tag` 时用 `\begin{equation}` 包裹），而非绕过 LaTeX。只有在 LaTeX 渲染确实失败时，才降级使用 MinerU 公式图片。

#### 1c. 用 Read 工具读取关键图片（必须）

**不要跳过这一步。** Claude 的 Read 工具支持多模态，能直接"看懂"图表内容。按 1b 分类结果，对**必须**和**重要**两类图执行 Read，记录：
- 展示了什么内容（架构？实验结果？消融对比？公式片段？）
- 对应论文 Figure X（编号）或公式编号
- 关键洞察是什么（为什么这张图/公式重要）

> 这一步的目的是确保你写的图片说明是**看图说话**，而非从论文文字中猜。图片的细节、趋势、对比关系只有通过"看"才能获取。

### 第三步：风格选择（询问用户）

在生成文章前，**必须询问用户**以下选项：

#### 1. 写作风格（必选）

| 风格 | 特点 | 适用场景 |
|------|------|----------|
| **storytelling**（故事型） | 从直觉出发，用比喻和例子，像讲故事 | 公众号、技术博客、科普 |
| **academic**（学术型） | 专业术语，严谨表述，保留原文概念 | 学术报告、论文综述、研究组分享 |
| **concise**（精炼型） | 直击核心，表格列表，信息密度高 | 快速了解、论文速览、技术调研 |

#### 2. 公式选项（可选）

| 选项 | 说明 |
|------|------|
| **with-formulas** | 插入公式图片并详细讲解符号含义 |
| **no-formulas**（默认） | 纯文字描述，不包含公式图片 |

#### 3. 代码选项（可选，仅当论文有 GitHub 时）

| 选项 | 说明 |
|------|------|
| **with-code** | 克隆仓库，贴关键源码，代码与论文对照讲解 |
| **no-code**（默认） | 不包含代码分析 |

**询问示例：**

> 请选择文章风格：
> 1. **academic** - 学术型，专业严谨（默认推荐）
> 2. **storytelling** - 故事型，朴素接地气
> 3. **concise** - 精炼型，快速阅读
>
> 是否需要公式讲解？（论文包含数学公式时推荐）
> 是否需要结合 GitHub 代码分析？（检测到开源仓库：xxx）

**如果用户不确定选哪个，默认使用 academic（学术型）风格。**

### 第四步：智能生成文章

根据用户选择的风格，阅读对应的风格定义文件：
- `styles/storytelling.md` - 故事型风格指南
- `styles/academic.md` - 学术型风格指南
- `styles/concise.md` - 精炼型风格指南
- `styles/with-formulas.md` - 公式讲解指南

#### 论文配图原则（必须遵守）

**论文配图是论文分析的必要组成部分，不可省略。** 架构图、实验结果、消融对比、公式片段都是论文的核心信息载体。纯文字解读 + "见图 3" 的文字引用对读者帮助有限。

- **≥3 张配图**：每篇分析文章至少嵌入 3 张关键图（架构图 + 主要实验结果 + 消融对比）。**公式片段不计入此配额**——它们是额外的重要素材
- **按 1b 分类策略决定**：架构图/方法图/实验结果图（必须）→ 全文嵌入；公式片段（重要）→ 选择性嵌入关键公式；表格图片（可选）→ 复杂表格保留图片，简单数据转 markdown
- **先读图再写文**：对每张图执行 Read（Claude 多模态），看懂内容后再写图片说明——不要从论文文字中猜测图表内容
- **策略性放置**：架构图放方法详解开头，实验结果图放对应实验段落，消融图放消融分析段落，公式图放对应公式讨论处
- **每张图配分析性说明**：不只是"Figure X: 架构图"，要写清楚图中展示了什么、关键信息在哪、读者应该注意什么
- **Figure 编号对应**：引用论文中的 Figure X 编号，方便读者对照原文

> 无配图的论文分析视为不完整。如确实无法获取配图（非 PDF 输入且 arxiv HTML 无图），必须在文章中标注原因。

#### 通用写作原则

**避免：**
- AI 常用词（"深入探讨"、"至关重要"、"在...领域"）
- 机械化章节标题
- LaTeX 公式语法（如 `$\mathcal{O}(1)$`）- 使用提取的公式图片
- 平铺直叙的技术描述

**采用：**
- 自然段落叙述
- 充分利用 MinerU 提取的图片
- 论文中的每张关键图都应该被讲解到
- 公式截图比 LaTeX 语法更易读

#### storytelling 风格方法论（故事型专用）

以下方法论仅在用户选择 **storytelling** 风格时应用：

**1. 从直觉切入，不要直接讲技术**
- 错误："本文提出了一种基于哈希表的条件记忆模块"
- 正确："你有没有想过，大模型其实是没有记忆功能的？"

**2. 先讲历史背景，再讲创新**
- 介绍新技术前，先解释相关的旧技术
- 让读者理解"为什么需要这个创新"

**3. 用简单例子贯穿全文**
- 选一个简单的例子反复使用
- 例如："中国的首都在北京"

**4. 使用生动的比喻**
- "大炮打蚊子"、"查字典 vs 背字典"
- 让抽象概念具象化

**5. 逻辑递进，层层深入**
- 简单问题 → 复杂问题 → 解决方案

**6. 提炼核心洞见**
- 用一句话总结，如"记忆归记忆，计算归计算"

#### 文章结构

**1. 论文信息**

```markdown
**论文标题**：xxx
**论文链接**：[arXiv](https://arxiv.org/abs/xxxx)
**作者团队**：xxx
```

**2. 直觉引入**（2-3段）
- 从一个问题或场景开始
- 让读者产生好奇心
- 引出"为什么需要这个研究"

**3. 背景知识**（3-4段）
- 解释相关的基础技术或历史方法
- 用简单例子说明
- 让读者理解现有方案的局限

**4. 核心创新**（4-5段）
- 详细讲解论文的创新点
- **架构图（Figure 1 类）必须放在本节开头**，让读者先有全局认知
- 每个创新点都要有图片支撑
- 用比喻和例子让抽象概念具象化
- 公式用图片展示，不用 LaTeX 语法

**5. 实验验证**（2-3段）
- **关键实验结果图（Figure N 类）必须嵌入对应段落**
- 对比分析和数据解读——不只是报数字，要解释"这意味着什么"
- 突出最亮眼的结果
- 实验数据表格要配解读

**6. 深入分析**（2-3段）
- 机制分析、消融实验等
- **消融对比图（ablation 类）放在消融实验段落**
- 解释"为什么这个方法有效"
- 提供更深层次的理解

**7. 思考与展望**（1-2段）
- 提炼核心洞见
- 预测未来发展方向
- 个人观点和评价

## 注意事项

- **优先使用 MinerU API**，精度最高，支持公式/表格
- **论文配图是必选项**：按 1b 分类策略决定哪些图嵌入——架构图/方法图/实验结果图（必须）、公式片段（重要）、表格图片（可选）
- **公式渲染注意**：`paper-to-wechat` 的 `render-formulas.py` 使用 `$\displaystyle$` 包裹公式，**不支持 `\tag{1}` 等 amsmath 标签**。如果 paper_analysis.html 中的 LaTeX 公式含 `\tag`，不要直接在 `$$...$$` 内写 `\tag{}`——改用 `\begin{equation}...\tag{1}\end{equation}` 包裹，或排版时去掉 `\tag{}` 保留公式本体、方程编号手动补为文字。**公式优先使用 LaTeX 代码渲染**，MinerU 公式图片仅在 LaTeX 渲染失败时作为备选
- **先读图再写文**：对每张图执行 Read（Claude 多模态），看懂内容后再写说明——不要从文字猜图表
- 不输出分析过程，用户只看最终文章
- 避免分点列表，使用自然段落叙述
- **公众号发布**：用 gzh-design（石墨极简风）排版 → paper-to-wechat skill 发布
